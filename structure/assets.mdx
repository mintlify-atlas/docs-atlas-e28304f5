## CSS architecture

The project uses a modular CSS architecture where `main.css` serves as the entry point and imports all other stylesheets.

### main.css

Location: `source/assets/css/main.css`

The main stylesheet that imports all CSS modules in a specific order:

```css title="source/assets/css/main.css"
@import url("./base.css");
@import url("./layout.css");
@import url("./components.css");
@import url("./sections.css");
@import url("./responsive.css");
```

**Import order matters**: Base styles load first, followed by layout, components, sections, and finally responsive overrides.

## JavaScript files

Currently the project has minimal JavaScript with only one file for interactive functionality.

### projects-filter.js

Location: `source/assets/js/projects-filter.js`

Handles the project filtering functionality on the projects page.

<Accordion title="How it works">
  The script listens for clicks on filter buttons and shows/hides project cards based on their `data-type` attribute.
  
  ```javascript
  document.addEventListener("DOMContentLoaded", () => {
    const filterButtons = document.querySelectorAll(".filter-btn");
    const projectCards = document.querySelectorAll(".project-item");
    
    filterButtons.forEach((button) => {
      button.addEventListener("click", () => {
        const selectedType = button.dataset.filter;
        
        // Update active button
        filterButtons.forEach((item) => item.classList.remove("active"));
        button.classList.add("active");
        
        // Show/hide cards
        projectCards.forEach((card) => {
          const type = card.dataset.type;
          const shouldShow = selectedType === "all" || selectedType === type;
          card.style.display = shouldShow ? "block" : "none";
        });
      });
    });
  });
  ```
</Accordion>

<Accordion title="Usage in projects.html">
  **Filter buttons:**
  ```html
  <button class="filter-btn active" data-filter="all">All</button>
  <button class="filter-btn" data-filter="residential">Residential</button>
  <button class="filter-btn" data-filter="commercial">Commercial</button>
  ```
  
  **Project cards:**
  ```html
  <article class="card project-item" data-type="residential">
    <!-- Project content -->
  </article>
  <article class="card project-item" data-type="commercial">
    <!-- Project content -->
  </article>
  ```
  
  **Script inclusion:**
  ```html
  <script src="../assets/js/projects-filter.js"></script>
  ```
</Accordion>

<Accordion title="Key features">
  - Only runs if both filter buttons and project cards are found on the page
  - Updates button active state on click
  - Filters cards by matching `data-filter` with `data-type`
  - "All" filter shows all cards
  - Uses simple `display: none/block` for show/hide functionality
</Accordion>

## Asset loading

### CSS loading

All pages load only `main.css`, which imports all other stylesheets:

```html
<!-- From index.html -->
<link rel="stylesheet" href="assets/css/main.css" />

<!-- From pages/ directory -->
<link rel="stylesheet" href="../assets/css/main.css" />
```

This ensures:
- Single HTTP request for all CSS
- Consistent styles across all pages
- Modular organization for development

### JavaScript loading

JavaScript is loaded only where needed:

```html
<!-- Only on projects.html -->
<script src="../assets/js/projects-filter.js"></script>
```

Script is placed at the end of the `<body>` to ensure DOM is loaded before execution.
